# Linux Docker Compose Deployment

This is the recommended all-in-one Linux deployment. It runs:

- `bridge`: MCQQ Bridge HTTP service, Web UI, SQLite, behavior-pack generator.
- `napcat`: NapCat Docker image for QQ login and OneBot v11.

NapCat Docker uses the official community image documented by `NapNeko/NapCat-Docker`: `mlikiowa/napcat-docker:latest`.

## Quick Start

```bash
cd deploy/compose
cp .env.example .env
vim .env
./init.sh
```

Edit at least:

```env
BRIDGE_PUBLIC_URL=http://YOUR_SERVER_IP:8080
QQ_GROUP_ID=123456789
```

If your server user is not UID/GID `1000`, also change:

```env
BRIDGE_UID=1000
BRIDGE_GID=1000
NAPCAT_UID=1000
NAPCAT_GID=1000
```

Then open:

```text
Bridge Web UI: http://YOUR_SERVER_IP:8080/setup
NapCat WebUI:  http://YOUR_SERVER_IP:6099/webui
```

Log in to QQ from NapCat WebUI, then install:

```text
packs/mcqq-bridge-behavior-pack.mcpack
```

into the BDS world behavior packs.

## Commands

```bash
./start.sh
./stop.sh
./restart.sh
./restart.sh bridge
./restart.sh napcat
./logs.sh
./logs.sh bridge
./logs.sh napcat
./doctor.sh
./generate-pack.sh
```

## Data Layout

```text
data/           Bridge config.yml and SQLite database
logs/           Bridge logs
packs/          Generated behavior packs
napcat/config/  NapCat OneBot config generated by Bridge
napcat/qq/      QQ login/session data
napcat/plugins/ NapCat plugins
```

These directories are runtime data and should not be committed.

## Config Without Web UI

```bash
docker compose exec bridge config show
docker compose exec bridge config set qq.group_id 123456789
docker compose exec bridge config set qq.forward_prefix ""
docker compose exec bridge config set server.public_url http://YOUR_SERVER_IP:8080
docker compose exec bridge pack generate /app/packs/mcqq-bridge-behavior-pack.mcpack
docker compose restart bridge
```

## Notes

- BDS must be able to reach `BRIDGE_PUBLIC_URL`.
- Do not use `127.0.0.1` for `BRIDGE_PUBLIC_URL` unless BDS is inside the same network namespace as Bridge.
- The NapCat WebUI port `6099` should be protected by firewall rules if exposed to the public internet.
